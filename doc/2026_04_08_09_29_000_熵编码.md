H264/265的熵编码采用cabac二进制上下文编码，需要一个bit一个bit的进行编码
AV1为了解决这个问题引入ANS编码，思路是期望能够提高前面的并行度，这是熵编码的核心问题和瓶颈

下面详细阐述H265/AV1的熵编码过程:  

**一、H265 CABAC**  

输入语法元素-->二值化(多种方案)-->根据语法元素判断是常规bin还是旁路bin编码码流-->更新上下文模型

语法元素根据类型分类，区分了是否依赖上下文的编码划分

**二值化方案**  
1、截断莱斯编码(Truncated Rice,TR)：多用于编码残差系数，通过一个门限值cMax和Rice参数cRiceParam来控制编码长度。(需要找到要编码的最大值)  
    (1)计算关键中间值 cTrMax = cMax >> cRiceParam (= floor(cMax/2^cRiceParam)) ,cTrMax 表示了前缀1中的最大个数  
       x 为计算待编码前缀时得到的值  
    (2)x < cMax的情况下，即未达到截断上限，采用1bin前缀+定长后缀的形式  
          计算前缀prefixVal = x >> cRiceParam，生成prefixval个1后跟一个0  
          如果为0则只有1bit0  
          后缀值为suffixVal = x & ((1 << cRiceParam - 1))即x的cRiceParam位  
          将suffixVal转换为长度为cRiceParam的定长串，高位补0  
    (3)x = cMax,即达到截断上限时，只生成前缀全是1，不跟0  
          前缀为cTrMax个1  
          后缀值为suffixVal = x - (cTrMax << cRiceParam)  
          将suffixVal转换为长度为cRiceParam的定长串，高位补0  
    因cRiceParam决定了编码过程，为了自适应，会动态确定其值，if absCoef > (3*(1 << cRiceParam)): cRiceParam = min(cRiceParam + 1,4)  

2、截断一元编码，即cRcieParam=0退化而成  
3、k阶指数哥伦布编码(k-th Order Exp-Golomb，EGK)：无截断限制，任意长度  
   前缀：由(x >> k) + 1个0后跟一个1组成，floor(x/2^k)+1
   后缀：x的低K位以定长形式附在末尾
   但实际上编码过程为对于k阶的要编码的x，x_adj = x + 2^k,
   前缀：len = floor(log2(x_adj)) 个 0 后跟一个1
   后缀：x_adj的低len作为后缀，不足len时高位补0

4、定长(Fixed-length FL)：直接将数值转化为固定长度的二进制

二值化硬件trick：   
1、egk的前缀为一元码，可以复用。  
2、egk的后缀为定长码，可以共享FL。  
3、tu的单独通路更小，比完整的tr通路更省电。  
4、通常TR负责小系数，EGK处理罕见的大系数  


**cabac编码流程**  
上下文建模：为bin选择概率模型  
MPS最大概率符号：表示在当前上下文条件下最可能出现的bin值  
pStateIdx：一个取值范围为0到63的整数值，用于查表确定LPS，最小概率符号的概率值，对应[ 0.01875,0.5 ]  
1、根据bin所在的语法元素、bin索引及邻域状态选择上下文索引  
   每一种SE使用自己的上下文，对于每个bin来说需要知道这是第几个bin。
   ctxIdx = 基地址 + 块大小特征 + 位置特征 + binIdx
   如果编码bin与MPS一致，则++pStateIdx，最高到63，如果与LPS一致则pStateIdx--最低为0，为0时翻转MPS/LPS

**ANS编码流程**
AV1采用rANS作为核心熵编码，一次编码2-16个符号
（编码的本质是在0-1区间内每编一次，把概率细分一次，根据最终的数字决定了整个信息，更新过程中细分到一定程度需要把高bit往外输出，从而继续细分范围，这就是range<256时需要输出的本质，log为下界其即起始位置）
state：编码器状态32btis，编码时增大，重归一化时减小。
CDF[ s ]:累积分布函数15bits(0-32767)，符号s的累积概率CDF[ M ] = 32768
freq[ s ]:符号频率15bits，CDF[ S ] = sum(freq[ 0...s-1 ])
M:符号数2-16，单步编码符号个数，由语法元素决定
F：频率总和32768，sum(freq[ 0..M-1 ]) = 32768
编码状态state = (state / freq[ s ]) * F + (state % freq[ s ]) + CDF[ s ]
重归一化:state超过阈值，右移吐出高bits

一、初始化：
1、初始化帧，根据元素选择对应的CDF表，确定M的值
2、state = 32768 ，选择窗口的中点
3、初始化CDF：CDF [ s ] = (32768 / M) * s, 每个上下文带计数最多32次，快速适应统计变化

二、编码单符号：
1、查CDF区间，找到满足CDF [ s ] ≤ x  < CDF [ s+1 ] 的区间
2、计算频率：freq[ s ] = CDF [ s+1 ] - CDF [ s ]
3、更新状态： div = state / freq[ s ], mod = state % freq[ s ], state = div * F + mod + CDF[ s ]
4、重归一化，若state ≥ 32768 << 16, 重复直到state < 32768 << 16。

三、概率模型更新（自适应）
编码后更新CDF保证压缩效率
1、对符号s的频率加步长避免溢出，freq [ s ] += 16; CDF [ s + 1.. M ] += 16;
2、归一化防止溢出：若sum(freq) > 32768,整体左移
3、饱和限制，同一上下文最多更新32次，后续用最终CDF，避免过拟合
4、收尾，输出剩余bits，全部符号编码完成后强制输出剩余的高位，完成拼接，每次取8bit进入码流直到state=0，写入结束标记0x80标识结束。

**硬件设计trick**
对于标准中当range < 256时 输出，因一周期左移较多bit对timing不算太友好，且大量情况下并非整个移动8bits，所以设计过程中可以切分为左移2bits，通过4级计算模块来拆解步骤以减少功耗，优化计算过程。
